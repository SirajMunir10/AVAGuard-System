# Troubleshooting: Windows Autopilot

**Domain:** Intune
**Subdomain:** Windows Autopilot
**Incident Type:** Troubleshooting

## Scenario / Query
How can ETW tracing be used to troubleshoot Windows Autopilot issues during OOBE?

## Remediation Steps
1. ETW tracing can be used to get detailed information from Windows Autopilot and related components.
2. The ETW trace files can be viewed using the Windows Performance Analyzer or similar tools.

## Validation
1. On the affected device, open an elevated PowerShell prompt and run: `Get-WinEvent -LogName Microsoft-Windows-User Device Registration/Admin | Select-Object -First 10` to confirm registration events are present.
2. Run: `Get-WinEvent -LogName Microsoft-Windows-DeviceManagement-Enterprise-Diagnostics-Provider/Admin | Select-Object -First 10` to verify Autopilot diagnostic events are logged.
3. Check that the ETW trace file (e.g., Autopilot.etl) was successfully generated by running: `Test-Path C:\ProgramData\Microsoft\DiagnosticLogCSP\Autopilot.etl`.
4. Open the .etl file in Windows Performance Analyzer and confirm that Autopilot-related events (e.g., TPM attestation, Azure AD join) are visible and contain no critical errors.

## Rollback
1. If ETW tracing is no longer needed, stop any active traces by running: `logman stop AutopilotTrace -ets` (if a custom trace was started).
2. Delete the generated .etl file: `Remove-Item C:\ProgramData\Microsoft\DiagnosticLogCSP\Autopilot.etl -Force`.
3. If the trace was started via the 'DiagnosticLogCSP' CSP, disable it by setting the policy to '0' or removing the configuration profile from Intune.
4. Reboot the device to ensure all tracing sessions are fully stopped and system resources are released.
