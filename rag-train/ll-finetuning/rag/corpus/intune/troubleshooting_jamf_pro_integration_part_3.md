# Troubleshooting: Jamf Pro Integration

**Domain:** Intune
**Subdomain:** Jamf Pro Integration
**Incident Type:** Troubleshooting

## Scenario / Query
Mac devices prompt for keychain sign-in when opening Microsoft 365 apps after configuring Intune and Jamf Pro integration with conditional access policies

## Environment Context
- **Tenant Type:** Microsoft Entra ID with Intune and Jamf Pro integration
- **Configuration:** Conditional access policies deployed for Jamf Pro managed devices

## Symptoms
- Users receive password prompts when opening Microsoft 365 applications such as Teams, Outlook, and other apps requiring Microsoft Entra authentication
- Prompt text similar to: 'Microsoft Teams wants to sign using key "Microsoft Workplace Join Key" in your keychain. To allow this, enter the "login" keychain password'

## Error Codes
N/A

## Root Causes
1. Prompts are generated by Jamf Pro for each applicable app that requires Microsoft Entra registration

## Remediation Steps
1. At the prompt, the user must provide their device password to sign in to Microsoft Entra ID
2. Options include: Deny - Do not sign in and do not use the app; Allow - A one time sign-in (next time the app opens, it prompts for sign-in again); Always Allow - The sign-in credentials are cached for the application (next time the app opens, it doesn't prompt for sign-in)
3. Selecting Always Allow for one app only approves that app for future sign-in; additional apps prompt for authentication until they also are set as Always Allow
4. Cached credentials for one app cannot be used by another app

## Validation
1. Open a Microsoft 365 app (e.g., Teams) on a Jamf-managed Mac. 2. When prompted, select 'Always Allow' and enter the device password. 3. Close the app and reopen it. Confirm no keychain prompt appears for that app. 4. Open a different Microsoft 365 app (e.g., Outlook). Verify a new keychain prompt appears for that app. 5. Select 'Always Allow' for the second app and confirm it no longer prompts on subsequent launches. 6. Check that the user can sign in to Microsoft Entra ID without errors.

## Rollback
1. Open Keychain Access (from /Applications/Utilities). 2. In the login keychain, locate and delete entries named 'Microsoft Workplace Join Key' or similar for the affected apps. 3. Clear the Microsoft 365 app cache by deleting the ~/Library/Group Containers/UBF8T346G9.Office folder. 4. Restart the Mac. 5. Reopen the Microsoft 365 apps and confirm the original keychain prompts return, requiring the user to re-authenticate.
