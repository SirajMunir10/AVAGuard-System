Amazon Detective is a security investigation service from AWS that ingests and analyzes log data from AWS resources to build a graph model of an environment. It is designed to help security engineers conduct faster and more effective investigations into security findings, such as those generated by Amazon GuardDuty, AWS Security Hub, or custom threat intelligence feeds. Unlike traditional log analysis tools that require manual querying across multiple sources, Detective automatically aggregates and correlates data from VPC Flow Logs, AWS CloudTrail, and Amazon EKS audit logs into a unified, interactive graph. This graph represents the relationships between AWS resources, users, and network traffic over time, enabling investigators to trace the root cause of an activity without writing complex queries.

**Key Components**

Amazon Detective operates on a few core architectural components that work together to provide investigative capabilities.

- **Behavior Graph**: This is the foundational data structure. Detective ingests raw logs and constructs a graph where nodes represent entities (e.g., EC2 instances, IAM users, S3 buckets) and edges represent interactions (e.g., network connections, API calls, authentication events). The graph is time-aware, meaning it stores historical snapshots of entity relationships. This allows an investigator to rewind to a specific point in time and see exactly which resources were communicating and what actions were taken.

- **Data Sources**: Detective currently ingests three primary data sources. VPC Flow Logs provide network traffic metadata (IP addresses, ports, protocols, packet counts). AWS CloudTrail records API activity for management events (e.g., CreateInstance, AssumeRole) and data events (e.g., GetObject on S3). Amazon EKS audit logs capture Kubernetes API server requests. All logs are normalized into a common schema, removing the need for the engineer to parse different log formats.

- **Findings and Alerts**: Detective does not generate its own security findings. Instead, it consumes findings from GuardDuty, Security Hub, or custom sources via the AWS Security Finding Format (ASFF). When a finding is ingested, Detective automatically enriches it with contextual data from the behavior graph. For example, a GuardDuty finding about a suspicious API call will be linked to the specific IAM user, the source IP address, and any related network flows.

- **Investigation Console**: The web-based console provides a visual interface for exploring the behavior graph. Key features include a timeline view that shows entity activity over a selected period, a resource map that displays connections between entities, and a search function that supports filtering by IP address, user name, resource ARN, or finding ID. The console also offers pre-built investigation playbooks for common scenarios like compromised credentials or data exfiltration.

- **API and SDK**: For automation, Detective exposes a set of APIs. Security engineers can programmatically retrieve graph data, list findings, and start investigations. This is critical for integrating Detective into a Security Information and Event Management (SIEM) system or a custom incident response workflow.

**Real-World Application**

Consider a scenario where GuardDuty generates a finding indicating that an EC2 instance is communicating with a known command-and-control (C2) IP address. A security engineer would use Detective to investigate.

1. **Initial Triage**: The engineer opens the Detective console and searches for the EC2 instance ID. The behavior graph immediately shows all network connections from that instance in the last 24 hours, including the suspicious C2 IP. The engineer can see the volume of traffic (bytes sent/received) and the ports used.

2. **Root Cause Analysis**: The engineer clicks on the C2 IP node in the graph. Detective reveals which IAM role the EC2 instance is using, what API calls that role made before the C2 communication started, and whether any other instances in the same VPC have similar connections. The timeline view shows that the instance launched a new process (visible via CloudTrail if Systems Manager Agent is configured) just before the C2 traffic began.

3. **Scope Determination**: Using the graph, the engineer identifies all resources that interacted with the compromised instance. This includes the security groups it belongs to, the subnets it resides in, and any S3 buckets it accessed. Detective shows that the instance read a sensitive configuration file from an S3 bucket two hours before the C2 traffic started.

4. **Remediation Guidance**: Based on the graph data, the engineer can isolate the instance by modifying its security group, revoking the IAM role’s permissions, and rotating the access keys used by the role. The timeline provides a precise window for log retention and forensic analysis.

**Security Relevance**

Amazon Detective addresses a critical gap in cloud security operations: the time and effort required to correlate disparate log sources during an incident. Without Detective, an engineer would need to manually query CloudTrail logs in Amazon Athena, VPC Flow Logs in Amazon S3, and EKS audit logs in CloudWatch Logs, then manually join the results. This process is error-prone and slow, especially under time pressure.

Detective’s graph model is particularly relevant for detecting lateral movement and privilege escalation. Because it tracks relationships over time, it can reveal that an IAM user who was granted temporary elevated privileges later accessed a sensitive database, even if the privilege escalation itself was not flagged as a finding. The service also supports cross-account investigations when enabled in an AWS Organizations setup, allowing a security team to trace activity across multiple accounts from a single console.

From a compliance perspective, Detective provides an auditable trail of investigation steps. Every action taken in the console or via the API is logged in CloudTrail. This is useful for demonstrating due diligence during audits or legal proceedings. Additionally, Detective retains behavior graph data for up to 365 days, enabling historical analysis for compliance reporting.

**Key Takeaways**
- Amazon Detective automates the correlation of VPC Flow Logs, CloudTrail, and EKS audit logs into a behavior graph, eliminating the need for manual log joins during investigations.
- The service enriches existing security findings from GuardDuty and Security Hub with contextual data, enabling faster root cause analysis and scope determination.
- Detective’s graph model and timeline view are essential for detecting lateral movement and privilege escalation that might be missed by signature-based detection alone.
