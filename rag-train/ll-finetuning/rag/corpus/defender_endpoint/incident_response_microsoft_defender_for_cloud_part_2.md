**Domain:** Defender for Endpoint

## Scenario / Query
How to respond to a security alert in Microsoft Defender for Cloud?

## Remediation Steps
1. Inspect resource context - sends you to the resource's activity logs that support the security alert
2. Mitigate the threat - provides manual remediation steps for this security alert
3. Prevent future attacks - provides security recommendations to help reduce the attack surface, increase security posture, and thus prevent future attacks
4. Trigger automated response - provides the option to trigger a logic app as a response to this security alert
5. Suppress similar alerts - provides the option to suppress future alerts with similar characteristics if the alert isn't relevant for your organization
6. For further details, contact the resource owner to verify whether the detected activity is a false positive
7. Investigate the raw logs generated by the attacked resource

## Validation
1. Confirm the security alert is visible in Microsoft Defender for Cloud: Navigate to Microsoft Defender for Cloud > Security alerts and verify the alert is listed with the expected severity and status. 2. Inspect resource context: Click on the alert and select 'Inspect resource context' to verify it opens the resource's activity logs and displays relevant entries. 3. Verify manual remediation steps: Follow the 'Mitigate the threat' link and confirm the provided steps are applicable and executed (e.g., check resource configuration or logs). 4. Check automated response: If a logic app was triggered, verify its run history in the Logic Apps designer to ensure it executed successfully. 5. Confirm suppression: If 'Suppress similar alerts' was used, verify the suppression rule is active under Security alerts > Suppression rules. 6. Validate raw logs: Access the attacked resource's diagnostic logs or Azure Monitor to confirm the raw logs show the expected activity.

## Rollback
1. If suppression was applied incorrectly: Navigate to Microsoft Defender for Cloud > Security alerts > Suppression rules, locate the rule, and delete or disable it to allow future alerts. 2. If a logic app was triggered in error: Go to the Logic App resource in Azure portal, review the run history, and disable or delete the logic app if it caused unintended actions. 3. If manual remediation steps caused issues: Reverse any configuration changes made (e.g., revert network security group rules, restore deleted resources from backup, or re-enable disabled services). 4. If the alert was dismissed prematurely: Reopen the alert from the Security alerts page by changing its status to 'Active' or re-evaluate the resource logs to confirm the threat. 5. For false positives: Contact the resource owner to verify the activity, and if confirmed, suppress future similar alerts as needed. 6. If raw logs were modified: Restore logs from backup or re-enable diagnostic settings to capture future logs.
