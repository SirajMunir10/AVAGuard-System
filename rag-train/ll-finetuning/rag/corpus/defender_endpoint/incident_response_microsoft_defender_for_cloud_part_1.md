# Incident Response: Microsoft Defender for Cloud

**Domain:** Defender for Endpoint
**Subdomain:** Microsoft Defender for Cloud
**Incident Type:** Incident Response

## Scenario / Query
How do I investigate a security alert in Microsoft Defender for Cloud?

## Environment Context
- **Tenant Type:** Azure
- **Configuration:** Microsoft Defender for Cloud enabled

## Symptoms
- Security alert received in Defender for Cloud

## Error Codes
N/A

## Root Causes
N/A

## Remediation Steps
1. Select an alert. A side pane opens and shows a description of the alert and all the affected resources.
2. Review the high-level information about the security alert: Alert severity, status, and activity time; Description that explains the precise activity that was detected; Affected resources; Kill chain intent of the activity on the MITRE ATT&CK matrix (if applicable).
3. Select View full details. The right pane includes the Alert details tab containing further details of the alert to help you investigate the issue: IP addresses, files, processes, and more.
4. Also in the right pane is the Take action tab. Use this tab to take further actions regarding the security alert. Actions such as: Inspect resource context - sends you to the resource's activity logs that support the security alert; Mitigate the threat - provides manual remediation steps for this security alert; Prevent future attacks - provides security recommendations to help reduce the attack surface, increase security posture, and thus prevent future attacks; Trigger automated response - provides the option to trigger a logic app as a response to this security alert; Suppress similar alerts - provides the option to suppress future alerts with similar characteristics if the alert isn't relevant for your organization.
5. For further details, contact the resource owner to verify whether the detected activity is a false positive. You can also investigate the raw logs generated by the attacked resource.

## Validation
1. Navigate to Microsoft Defender for Cloud in the Azure portal. 2. Under 'Security alerts', select the alert that was investigated. 3. Confirm the side pane opens and displays: alert severity, status, activity time, description, affected resources, and MITRE ATT&CK intent (if applicable). 4. Select 'View full details' and verify the Alert details tab shows IP addresses, files, processes, and other relevant data. 5. In the Take action tab, confirm the presence of options: Inspect resource context, Mitigate the threat, Prevent future attacks, Trigger automated response, and Suppress similar alerts. 6. Verify that the resource owner has been contacted (if applicable) and raw logs have been reviewed.

## Rollback
1. If the investigation reveals no threat or a false positive, in the Take action tab, select 'Suppress similar alerts' to prevent future alerts with similar characteristics. 2. If automated response was triggered, disable or modify the logic app from the Trigger automated response option. 3. If manual remediation steps were taken (e.g., Mitigate the threat), reverse those steps according to the resource's standard operating procedures. 4. If resource context was inspected, no rollback is needed; simply close the activity logs. 5. Document the false positive and notify the resource owner to disregard the alert.
