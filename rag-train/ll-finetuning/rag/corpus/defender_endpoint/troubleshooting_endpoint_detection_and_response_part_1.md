# Troubleshooting: Endpoint Detection and Response

**Domain:** Defender for Endpoint
**Subdomain:** Endpoint Detection and Response
**Incident Type:** Troubleshooting

## Scenario / Query
How to confirm automatic device isolation and troubleshoot failed or pending isolation actions in Microsoft Defender for Endpoint?

## Environment Context
- **Tenant Type:** Microsoft Defender for Endpoint
- **Configuration:** Automatic attack disruption enabled

## Symptoms
- Device isolation action shows as failed or pending
- Device appears isolated but investigation data cannot be collected

## Error Codes
N/A

## Root Causes
1. Device is offline and cannot report to Defender for Endpoint
2. Investigation method (e.g., live response) is not supported for that device or scenario
3. Required service endpoints are not reachable in the network configuration

## Remediation Steps
1. Open the relevant incident generated by automatic attack disruption in the Microsoft Defender portal
2. Review the Activity tab or Action center to see which automated response actions were applied
3. Open the affected device page and confirm that the device status shows that it's isolated
4. If the isolation action shows as failed or pending, confirm that the device is online and can report to Defender for Endpoint
5. Retry from the device action panel if available
6. If a device appears isolated but you can't collect investigation data, verify that your investigation method (for example, live response) is supported for that device and scenario
7. Confirm required service endpoints are reachable in your network configuration

## Validation
Open the affected device page and confirm that the device status shows that it's isolated
