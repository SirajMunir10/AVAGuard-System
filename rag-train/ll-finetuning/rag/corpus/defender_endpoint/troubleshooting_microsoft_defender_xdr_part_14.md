# Troubleshooting: Microsoft Defender XDR

**Domain:** Defender for Endpoint
**Subdomain:** Microsoft Defender XDR
**Incident Type:** Troubleshooting

## Scenario / Query
How to use the Dynamic Threat Detection Agent in Microsoft Security Copilot to triage alerts and uncover hidden threats across Defender and Microsoft Sentinel environments?

## Environment Context
- **Tenant Type:** Microsoft 365 Defender
- **Configuration:** Microsoft Security Copilot enabled with Dynamic Threat Detection Agent

## Symptoms
- Hidden threats not detected by standard alerting mechanisms
- False negatives in alert correlation

## Error Codes
N/A

## Root Causes
1. Gaps in alert correlation across Defender and Microsoft Sentinel environments

## Remediation Steps
1. Use the Dynamic Threat Detection Agent, an always on, adaptive backend service that uncovers hidden threats across Defender and Microsoft Sentinel environments
2. The agent uses AI to identify gaps and uncover false negatives by correlating alerts, events, anomalies, and threat intelligence
3. When the agent identifies a gap, it generates a dynamic alert with full context in the alert details, including natural language explanations, mapped MITRE ATT&CK techniques, and tailored remediation steps

## Validation
1. In Microsoft Security Copilot, open the Dynamic Threat Detection Agent dashboard and confirm the agent status is 'Active' and 'Always on'. 2. Review the 'Dynamic alerts' section to verify that at least one dynamic alert has been generated in the last 24 hours. 3. Open a dynamic alert and confirm it includes: a natural language explanation, mapped MITRE ATT&CK techniques, and tailored remediation steps. 4. Cross-check the alert details against Microsoft Defender and Microsoft Sentinel to ensure the alert correlates events from both environments. 5. Run a test query in Microsoft Sentinel (e.g., SecurityAlert | where AlertName contains 'Dynamic') to confirm the alert appears in Sentinel.

## Rollback
1. If the Dynamic Threat Detection Agent is causing performance issues or false positives, disable the agent in Microsoft Security Copilot settings by toggling 'Dynamic Threat Detection Agent' to Off. 2. Remove any dynamic alerts that were incorrectly generated by manually closing them in the Microsoft Defender portal (select alert -> 'Manage alert' -> 'Resolve'). 3. If the agent was enabled via a policy, revert the policy to its previous state (e.g., set 'EnableDynamicThreatDetection' to False). 4. Verify that standard alerting mechanisms are functioning by checking a known test alert in both Defender and Sentinel. 5. Monitor for 24 hours to ensure no new dynamic alerts are generated and that existing alert correlation returns to baseline behavior.
