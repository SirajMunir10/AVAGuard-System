# Hardening: Audit Log

**Domain:** Purview
**Subdomain:** Audit Log
**Incident Type:** Hardening

## Scenario / Query
How to monitor and audit Universal Print configuration changes for security compliance?

## Environment Context
- **Tenant Type:** Microsoft 365
- **Configuration:** Universal Print service enabled

## Symptoms
- Unauthorized changes to printer shares or access controls
- Unexpected modifications to tenant-level print settings

## Error Codes
N/A

## Root Causes
1. Lack of auditing for Universal Print activities
2. Insufficient monitoring of privileged actions

## Remediation Steps
1. Enable audit log search in the Microsoft 365 compliance center
2. Regularly review audit log entries for activities such as ShareAccessControlGroupAdded, ShareAccessControlUserAdded, ShareAccessControlAllowAllEnabled, ShareAccessControlAllowAllDisabled, TenantPrintSettingsModified, ConnectorRegistered, PrinterRegistered, PrinterSwapped
3. Set up alerts for critical activities like ShareDeleted, PullPrintPrinterDeleted, or TenantPrintSettingsModified

## Validation
1. Navigate to Microsoft 365 Defender > Audit > Search. 2. Run a search for the date range covering the remediation period. 3. Filter by workload 'Universal Print' and verify that entries for activities such as ShareAccessControlGroupAdded, ShareAccessControlUserAdded, ShareAccessControlAllowAllEnabled, ShareAccessControlAllowAllDisabled, TenantPrintSettingsModified, ConnectorRegistered, PrinterRegistered, PrinterSwapped appear. 4. Confirm that alerts for ShareDeleted, PullPrintPrinterDeleted, TenantPrintSettingsModified are configured and triggered. 5. Use PowerShell: Search-UnifiedAuditLog -StartDate (Get-Date).AddDays(-7) -EndDate (Get-Date) -Operations 'ShareAccessControlGroupAdded','ShareAccessControlUserAdded','ShareAccessControlAllowAllEnabled','ShareAccessControlAllowAllDisabled','TenantPrintSettingsModified','ConnectorRegistered','PrinterRegistered','PrinterSwapped','ShareDeleted','PullPrintPrinterDeleted' | Format-Table CreationDate,Operations,UserIds,ResultStatus -AutoSize

## Rollback
1. If audit log search was not previously enabled, disable it by navigating to Microsoft 365 Defender > Audit > Search and toggling off 'Audit log search'. 2. Remove any custom alert policies created for Universal Print activities via Microsoft 365 Defender > Alerts > Alert policies. 3. If PowerShell was used to enable auditing, run: Set-AdminAuditLogConfig -UnifiedAuditLogIngestionEnabled $false. 4. Confirm that no audit log entries for Universal Print are being generated by running: Search-UnifiedAuditLog -StartDate (Get-Date).AddDays(-1) -Operations '*' | Where-Object {$_.Workload -eq 'UniversalPrint'} | Format-Table CreationDate,Operations,UserIds
