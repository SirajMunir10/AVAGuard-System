# Troubleshooting: Audit Log

**Domain:** Purview
**Subdomain:** Audit Log
**Incident Type:** Troubleshooting

## Scenario / Query
How can I distinguish system-generated FileAccessed events from user-initiated FileAccessed events in the audit log, particularly those related to Insider Risk Management?

## Environment Context
- **Tenant Type:** Microsoft 365
- **Configuration:** Insider Risk Management with Content Explorer enabled

## Symptoms
- FileAccessed audit events appear in the audit log without a direct user file-open action
- Events are generated by Insider Risk Management/Content Explorer workflow

## Error Codes
N/A

## Root Causes
1. Creating a case in Insider Risk Management and enabling Content Explorer generates and records a FileAccessed event in the audit log
2. The event is generated by the Insider Risk Management/Content Explorer workflow rather than by a direct user file-open action

## Remediation Steps
1. Use the ApplicationId field to identify or filter these specific events when reviewing audit log results
2. Look for ApplicationId value 92876b03-76a3-4da8-ad6a-0511ffdf8647 to distinguish system-generated FileAccessed events from other FileAccessed activity

## Validation
Run the following command in the Microsoft 365 Purview compliance portal or via Search-UnifiedAuditLog in Exchange Online PowerShell to confirm that FileAccessed events with ApplicationId 92876b03-76a3-4da8-ad6a-0511ffdf8647 are present and correctly filtered:

Search-UnifiedAuditLog -StartDate (Get-Date).AddDays(-7) -EndDate (Get-Date) -Operations FileAccessed -Formatted | Where-Object {$_.ApplicationId -eq '92876b03-76a3-4da8-ad6a-0511ffdf8647'} | Format-Table CreationTime, UserIds, Operations, ApplicationId

Verify that the returned events correspond to Insider Risk Management/Content Explorer activity and not to direct user file-open actions. Additionally, confirm that no other FileAccessed events with a different ApplicationId are incorrectly included.

## Rollback
If the filtering causes missing legitimate user-initiated FileAccessed events or other issues, remove the ApplicationId filter from your audit log queries and revert to the default search behavior. To restore the original audit log review process, run:

Search-UnifiedAuditLog -StartDate (Get-Date).AddDays(-7) -EndDate (Get-Date) -Operations FileAccessed -Formatted | Format-Table CreationTime, UserIds, Operations

If you have applied any custom audit log retention policies or views that rely on this filter, delete or disable those policies. No other rollback actions are required as the remediation only involves filtering logic and does not modify any system settings.
