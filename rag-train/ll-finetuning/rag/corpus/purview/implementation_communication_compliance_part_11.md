# Implementation: Communication Compliance

**Domain:** Purview
**Subdomain:** Communication Compliance
**Incident Type:** Implementation

## Scenario / Query
How do I provide feedback on a Copilot-generated summary in Communication Compliance?

## Environment Context
- **Tenant Type:** Microsoft 365
- **Configuration:** Copilot in Microsoft Purview

## Symptoms
N/A

## Error Codes
N/A

## Root Causes
N/A

## Remediation Steps
1. Select the drop-down arrow in the lower-right corner below the summary
2. Enter your feedback

## Validation
1. Navigate to Microsoft Purview compliance portal > Communication Compliance > select a policy > open a message with a Copilot-generated summary. 2. Confirm the drop-down arrow in the lower-right corner below the summary is present and clickable. 3. Click the arrow and verify the feedback options (e.g., 'Good' or 'Bad') appear. 4. Select a feedback option and confirm the feedback is submitted (e.g., a confirmation message or visual indicator). 5. Repeat for at least two different messages to ensure consistency.

## Rollback
1. If the feedback submission fails or causes unexpected behavior, refresh the browser page and retry the feedback process. 2. If the issue persists, clear the browser cache and cookies, then sign out and sign back into the Microsoft 365 tenant. 3. If the drop-down arrow is missing or unresponsive, verify that the user has the required Communication Compliance role (e.g., Communication Compliance Admin, Analyst, Investigator) and that Copilot in Microsoft Purview is enabled in the tenant. 4. As a last resort, contact Microsoft Support to report the issue and request a rollback of any recent configuration changes related to Copilot or Communication Compliance.
