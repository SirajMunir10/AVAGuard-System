## Remediation Steps
1. Use Microsoft Purview to manage data security and compliance for Microsoft 365 Copilot and Microsoft 365 Copilot Chat.
2. For more information, see 'Use Microsoft Purview to manage data security & compliance for Microsoft 365 Copilot & Microsoft 365 Copilot Chat'.

## Validation
1. Confirm that audit log search is enabled in the Microsoft Purview compliance portal: navigate to Audit > Audit log search and verify that 'Start recording user and admin activity' is set to On. 2. Run a sample audit log search for Copilot activities: use Search-UnifiedAuditLog -Operations 'CopilotInteraction' and verify that results are returned. 3. Check that audit retention policies are applied to Copilot data: run Get-AuditLogRetentionPolicy and confirm a policy exists for 'Copilot' workloads. 4. Validate that the required audit records are being generated by performing a test Copilot query and then searching the audit log for that specific event within 24 hours.

## Rollback
1. If audit log search was enabled and causes performance issues, disable it: navigate to Audit > Audit log search and set 'Start recording user and admin activity' to Off. 2. If audit retention policies were created for Copilot, remove them: run Remove-AuditLogRetentionPolicy -Identity 'CopilotRetentionPolicy'. 3. If any custom audit log search queries were created for Copilot, delete them: navigate to Audit > Audit log search > Saved queries, select the query, and delete it. 4. If changes were made to audit log settings via PowerShell, revert using Set-AdminAuditLogConfig -UnifiedAuditLogIngestionEnabled $false.
