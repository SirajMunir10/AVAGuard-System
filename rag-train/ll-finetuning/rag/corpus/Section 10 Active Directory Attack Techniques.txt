================================================================================
SECTION 10: ACTIVE DIRECTORY ATTACK TECHNIQUES
================================================================================

Active Directory is the most frequently targeted component in enterprise network penetration tests because compromising it grants control over every domain-joined system and user account in the organization. A complete internal penetration test must include comprehensive Active Directory security assessment.

Kerberoasting is one of the most common and effective Active Directory attack techniques. It exploits the fact that any authenticated domain user can request Kerberos service tickets for accounts configured with a Service Principal Name. These tickets are encrypted with the service account's password hash and can be extracted from memory and cracked offline without generating failed authentication events. Accounts with weak passwords are typically cracked within hours using GPU-accelerated password cracking tools. The attack is performed using tools like Impacket's GetUserSPNs.py or Rubeus, and findings should document exactly which service accounts were crackable and what level of access their credentials provide.

AS-REP Roasting targets domain accounts configured with the Do not require Kerberos preauthentication attribute, which is sometimes set on service accounts or legacy application accounts. Without preauthentication required, an attacker can request an authentication response for any such account without supplying credentials. The response contains data encrypted with the account's password hash and can be cracked offline. This is performed with Impacket's GetNPUsers.py and should be tested against all domain accounts during an internal assessment.

Pass-the-Hash allows an attacker to authenticate to Windows services using a captured NTLM password hash without knowing the underlying plaintext password. This is possible because the Windows authentication protocol accepts the hash directly as proof of identity. NTLM hashes are captured from LSASS process memory, the SAM database, or from NTDS.dit on a domain controller. CrackMapExec automates Pass-the-Hash attacks against entire subnets, allowing rapid identification of systems where a captured hash grants local administrator access.

DCSync is an advanced technique that simulates the replication behavior of a domain controller to extract password hashes from Active Directory. An attacker with an account that has the Replicating Directory Changes and Replicating Directory Changes All permissions, which are granted to Domain Admins, can use Mimikatz to request replication of all password hashes from the domain without logging onto the domain controller itself. This effectively provides the attacker with every user's credentials in the domain. The KRBTGT hash retrieved through DCSync enables Golden Ticket creation for indefinite persistent access.

Golden Ticket attacks use the KRBTGT account hash to forge Kerberos ticket granting tickets for any user in the domain, including accounts that do not exist. Forged tickets are valid for ten years by default and survive password resets unless the KRBTGT password is reset twice. A tester who achieves a Golden Ticket has demonstrated complete and persistent domain compromise. This should always be documented as a critical finding with the specific attack path that led to KRBTGT hash acquisition.

BloodHound is an essential tool for Active Directory penetration testing that uses graph theory to map attack paths through the domain. SharpHound collects domain information including group memberships, access control lists, session data, and trust relationships. BloodHound visualizes this data and can automatically identify the shortest attack path from any starting user to Domain Admin. Paths that traverse misconfigurations such as a user having GenericWrite permission over a group that is a member of a privileged group, or a computer account with delegation configured, can lead to privilege escalation that would be impossible to identify through manual review alone.
