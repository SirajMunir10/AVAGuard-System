================================================================================
SECTION 6: ACTIVE RECONNAISSANCE
================================================================================

Active reconnaissance involves directly interacting with target systems to gather information. Unlike passive recon, active recon is detectable and must only be performed after written authorization has been confirmed. Active recon generates network traffic, may appear in server logs, and could trigger intrusion detection systems. It provides more precise and actionable intelligence than passive methods alone.

Port scanning is the starting point for active reconnaissance against network targets. A comprehensive scan of all 65,535 TCP ports and common UDP ports using a tool like nmap identifies every listening service on in-scope hosts. Service version detection flags such as -sV cause nmap to probe each open port and attempt to identify the exact software and version running on it. Operating system detection using -O can reveal the underlying platform. The full command for a thorough initial scan of a target host would typically be nmap -sV -sC -O -p- targeting the in-scope IP range, saving output in all formats for later reference.

Service fingerprinting extends port scanning results by cross-referencing identified service versions against vulnerability databases. Every service version banner should be looked up in the National Vulnerability Database, ExploitDB, and Vulners to identify known CVEs. A service running an outdated version of OpenSSL, Apache Struts, or a popular VPN client may have publicly available exploit code that can lead directly to remote code execution. Version banners that reveal a product in its end-of-life period represent systemic risk beyond any single CVE.

Web application fingerprinting identifies the technology stack, content management system, and security controls protecting a web application before testing begins. Tools like WhatWeb and Wappalyzer analyze HTTP response headers, JavaScript file paths, cookie names, HTML comments, and page content to identify frameworks, libraries, and platforms. The wafw00f tool specifically detects web application firewall signatures, which is important for planning bypass techniques during exploitation. Knowing the CMS version before testing allows the tester to search for known vulnerabilities specific to that version rather than testing generically.

LDAP and Active Directory enumeration from an unauthenticated or low-privileged position can reveal significant information about the internal structure of the organization. Tools like ldapsearch and enum4linux-ng can enumerate domain users, groups, organizational units, password policies, and computer accounts even with anonymous or null session binds in misconfigured environments. This information directly enables attacks like Kerberoasting and AS-REP roasting later in the engagement.

SNMP enumeration targets network devices that have SNMP enabled with default or weak community strings. The SNMP protocol exposes a management information base containing detailed configuration data including interface addresses, routing tables, installed software, and system descriptions. Brute-forcing common community strings such as public, private, and community using tools like onesixtyone can yield significant network topology intelligence from routers, switches, and printers without requiring any credentials.
