**Definition**

Business-to-Business (B2B) collaboration in Microsoft Entra ID allows you to securely share your organization's applications and services with guest users from other organizations. Guest users are external users who sign in using their own work, school, or social identities. Business-to-Consumer (B2C) is a separate service for customer-facing identity management, but in the context of Entra ID, B2B guest access is the primary mechanism for external collaboration. Security for guest access involves controlling what guests can see, how they authenticate, and how they are governed.

**Components**

B2B guest access involves several components. Invitation process: you can invite guests via email, direct link, or self-service sign-up. Guest user object: each guest has a user object in your Entra ID tenant with a UserType of "Guest". External identities settings: under Entra ID > External Identities > External collaboration settings, you can restrict guest invite permissions (e.g., only admins can invite) and set guest access levels (e.g., limited to properties of their own profile). Cross-tenant access settings: under External Identities > Cross-tenant access settings, you can configure inbound and outbound trust settings for specific tenants, including MFA trust, device trust, and user attribute mapping. Conditional Access for guests: you can apply Conditional Access policies to guest users, such as requiring MFA for all external access or blocking access from untrusted locations. Access reviews: periodic reviews of guest access to ensure only necessary guests retain access.
