================================================================================
SECTION 9: NETWORK PENETRATION TESTING - THE KILL CHAIN
================================================================================

Network penetration testing follows a structured attack chain that mirrors how real threat actors progress from initial access to their objectives. Understanding and documenting each step of this chain is essential for producing findings that demonstrate realistic impact rather than isolated technical issues.

Initial access is the first stage of the kill chain, representing how an attacker first gains a foothold in the target environment. Common initial access vectors include exploiting vulnerabilities in internet-facing services such as VPN appliances, remote desktop services, and web applications, exploiting unpatched software with known CVEs, phishing employees to harvest credentials or deliver malware, and obtaining valid credentials from breach databases and using them to authenticate to exposed services. The initial access technique used in a penetration test should be documented along with its MITRE ATT&CK technique identifier, such as T1190 for exploiting public-facing applications or T1078 for valid accounts.

Execution covers how the attacker runs malicious code after gaining initial access. Living-off-the-land techniques use legitimate system utilities already present on the target to execute commands, avoiding detection by endpoint security tools. On Windows systems this includes PowerShell, Windows Management Instrumentation, certutil, mshta, and regsvr32. On Linux systems it includes bash, python, curl, and scheduled cron jobs. Web shells planted in web application directories are a common execution mechanism following exploitation of a web vulnerability.

Persistence techniques ensure the attacker maintains access to the environment even after a reboot or password change. On Windows, common persistence mechanisms include scheduled tasks, registry run keys in HKCU and HKLM hives, Windows services, startup folder entries, and WMI event subscriptions. On Linux, common methods include cron jobs, systemd service units, SSH authorized_keys modifications, and bashrc or profile modifications. Persistence mechanisms should be documented precisely and removed at engagement conclusion unless the client has explicitly requested they be left for post-engagement validation testing.

Privilege escalation moves the attacker from a low-privilege account to a high-privilege account or system context. On Windows, common techniques include exploiting weak service permissions where an attacker can replace a service binary, unquoted service path vulnerabilities, AlwaysInstallElevated policy misconfigurations, token impersonation using tools like the Potato family of exploits, and kernel exploits targeting unpatched local privilege escalation CVEs. On Linux, privilege escalation commonly exploits SUID binaries, writable cron jobs running as root, sudo misconfigurations, NFS no_root_squash mounts, and kernel exploits.

Defense evasion is the set of techniques attackers use to avoid detection by antivirus software, endpoint detection and response tools, and SIEM monitoring. During a penetration test, demonstrating defense evasion capability is important for assessing the real-world risk of the findings. Common evasion techniques include Antimalware Scan Interface bypass to prevent PowerShell from being analyzed, Event Tracing for Windows patching to suppress telemetry, process injection to execute code within the memory space of legitimate processes, and encoding or obfuscating payloads to avoid signature-based detection.

Lateral movement is the process of using a foothold in one system to gain access to other systems within the environment. In Windows environments, lateral movement commonly leverages Pass-the-Hash using captured NTLM hashes to authenticate to other systems, Pass-the-Ticket using forged or stolen Kerberos tickets, Windows Remote Management, PsExec over SMB, and DCOM-based execution. In Linux environments, SSH key reuse and pivoting through compromised hosts via SOCKS proxies are the primary lateral movement mechanisms. BloodHound with SharpHound data collection should be used to map the attack paths available within Active Directory and identify the shortest path to domain compromise.

Exfiltration demonstrates the attacker's ability to remove sensitive data from the environment. During testing, exfiltration should be demonstrated using a small sample of non-sensitive placeholder data rather than real customer or employee information. The test should confirm whether the organization's data loss prevention controls would detect the transfer, whether encrypted channels such as DNS tunneling or HTTPS to a cloud service can bypass controls, and whether the volume and speed of transfer that would constitute a meaningful breach would trigger any alerts.
