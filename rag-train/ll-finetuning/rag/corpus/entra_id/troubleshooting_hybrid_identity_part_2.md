# Troubleshooting: Hybrid Identity (AttributeValueMustBeUnique)

**Domain:** Entra ID
**Subdomain:** Hybrid Identity
**Incident Type:** Troubleshooting

## Scenario / Query
How to resolve the AttributeValueMustBeUnique sync error when two objects have the same proxyAddresses or userPrincipalName value in Microsoft Entra ID?

## Environment Context
- **Tenant Type:** Microsoft Entra ID with hybrid sync
- **Configuration:** Microsoft Entra Connect sync

## Symptoms
- Sync operation fails with AttributeValueMustBeUnique error
- Two objects with different sourceAnchor (immutableId) attributes have the same value for proxyAddresses or userPrincipalName

## Error Codes
- `AttributeValueMustBeUnique`

## Root Causes
1. Microsoft Entra schema does not allow two or more objects to have the same value of proxyAddresses or userPrincipalName attributes
2. A duplicate value is assigned to an already synced object, which conflicts with another synced object

## Remediation Steps
1. Identify the duplicated proxyAddresses, userPrincipalName, or other attribute value that's causing the error
2. Identify which two or more objects are involved in the conflict using the report generated by Microsoft Entra Connect Health for sync
3. Identify which object should continue to have the duplicated value and which object shouldn't
4. Remove the duplicated value from the object that should not have that value
5. Make the change in the directory from where the object is sourced
6. In some cases, you might need to delete one of the objects in conflict
7. If you made the change in on-premises Active Directory, let Microsoft Entra Connect Sync the change for the error to get fixed
