# Troubleshooting: Microsoft Entra Connect Sync (ObjectTypeMismatch)

**Domain:** Entra ID
**Subdomain:** Microsoft Entra Connect Sync
**Incident Type:** Troubleshooting

## Scenario / Query
How to resolve an ObjectTypeMismatch sync error when Microsoft Entra ID attempts to soft match two objects of different types (e.g., user, group, contact) with the same proxyAddresses attribute value?

## Environment Context
- **Tenant Type:** Microsoft Entra ID (Azure AD) with hybrid identity
- **Configuration:** Microsoft Entra Connect sync with soft matching enabled

## Symptoms
- Microsoft Entra Connect sync operation results in an ObjectTypeMismatch sync error
- Two objects of different object type (e.g., user, group, or contact) have the same value for the proxyAddresses attribute

## Error Codes
- `ObjectTypeMismatch`

## Root Causes
1. Two objects of different object type (e.g., user, group, or contact) have the same value for the proxyAddresses attribute
2. Duplication of attributes used for soft matching is not permitted in Microsoft Entra ID

## Remediation Steps
1. Identify the duplicated proxyAddresses (or other attribute) value that is causing the error
2. Identify which two or more objects are involved in the conflict using the report generated by Microsoft Entra Connect Health for sync
3. Identify which object should continue to have the duplicated value and which object should not
4. Remove the duplicated value from the object that should not have that value
5. Make the change in the directory where the object is sourced from
6. In some cases, delete one of the objects in conflict
7. If the change was made in on-premises Active Directory, let Microsoft Entra Connect sync the change

## Validation
The sync error report in Microsoft Entra Connect Health for sync is updated every 30 minutes and includes the errors from the latest synchronization attempt
