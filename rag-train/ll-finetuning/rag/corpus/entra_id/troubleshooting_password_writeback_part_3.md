# Troubleshooting: Password Writeback (6800)

**Domain:** Entra ID
**Subdomain:** Password Writeback
**Incident Type:** Troubleshooting

## Scenario / Query
How to resolve password writeback error 6800 indicating the password reset service doesn't start on-premises?

## Environment Context
- **Tenant Type:** Microsoft Entra ID
- **Configuration:** Password writeback enabled on Microsoft Entra Connect

## Symptoms
- Error 6800 appears in the Microsoft Entra Connect machine's Application event log
- Federated, pass-through authentication, or password-hash-synchronized users can't reset their passwords

## Error Codes
- `6800`

## Root Causes
1. The WCF endpoint for password writeback failed to start during restart of the Azure AD Sync (ADSync) service

## Remediation Steps
1. Review event log errors generated by the PasswordResetService component to identify why the endpoint can't start up
2. Try to restart Microsoft Entra Connect if password writeback still isn't working
3. If the problem persists, try to disable and then re-enable password writeback
