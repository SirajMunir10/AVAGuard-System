# Troubleshooting: Microsoft Entra Connect Sync (InvalidSoftMatch)

**Domain:** Entra ID
**Subdomain:** Microsoft Entra Connect Sync
**Incident Type:** Troubleshooting

## Scenario / Query
How to fix the InvalidSoftMatch error during Microsoft Entra Connect synchronization?

## Environment Context
- **Tenant Type:** Microsoft Entra ID (Azure AD)
- **Configuration:** Microsoft Entra Connect Sync with on-premises Active Directory

## Symptoms
- InvalidSoftMatch error during synchronization
- Duplicate proxyAddresses, userPrincipalName, or other attribute values
- Two or more objects involved in a conflict

## Error Codes
- `InvalidSoftMatch`

## Root Causes
1. Duplicate proxyAddresses, userPrincipalName, or other attribute value causing the error
2. Microsoft Entra Connect may calculate a different value of the sourceAnchor attribute for an Active Directory object that represents the same entity (same user, group, or contact) as an existing Microsoft Entra object
3. Microsoft Entra Connect not configured with certain scenarios in mind, leading to ImmutableId attribute changes

## Remediation Steps
1. Identify the duplicated proxyAddresses, userPrincipalName, or other attribute value that's causing the error
2. Identify which two or more objects are involved in the conflict using the report generated by Microsoft Entra Connect Health for sync
3. Determine which object should continue to have the duplicated value and which object should not
4. Remove the duplicated value from the object that should not have that value
5. Make the change in the directory from where the object is sourced (e.g., on-premises Active Directory)
6. In some cases, delete one of the objects in conflict
7. If the change was made in on-premises Active Directory, let Microsoft Entra Connect Sync the change
