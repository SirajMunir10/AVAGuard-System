# Implementation: Incident Investigation

**Domain:** Sentinel
**Subdomain:** Incident Investigation
**Incident Type:** Implementation

## Scenario / Query
How do I create advanced analytics rules in Microsoft Sentinel to generate incidents for investigation?

## Environment Context
- **Tenant Type:** Azure
- **Configuration:** Microsoft Sentinel in the Azure portal, Analytics page

## Symptoms
N/A

## Error Codes
N/A

## Root Causes
N/A

## Remediation Steps
1. Connect data sources to Microsoft Sentinel
2. Create advanced analytics rules in the Analytics page that generate incidents
3. Assign and investigate incidents

## Validation
1. In the Azure portal, navigate to Microsoft Sentinel > Analytics. 2. Confirm that the rule you created appears in the list of active rules with status 'Enabled'. 3. Select the rule and review its configuration to ensure the rule logic, entity mappings, and incident creation settings are correct. 4. Use the 'Run Query' button in the rule details to verify the query returns expected results. 5. Check the Incidents page (Microsoft Sentinel > Incidents) to confirm that incidents are being generated from the rule's alerts. 6. Optionally, use Log Analytics to query SecurityAlert for alerts generated by the rule.

## Rollback
1. In the Azure portal, navigate to Microsoft Sentinel > Analytics. 2. Locate the rule you created. 3. Select the rule and click 'Disable' to stop it from generating new incidents. 4. If the rule is causing issues, you can delete it by selecting 'Delete' after disabling. 5. To remove any incidents already created, go to Microsoft Sentinel > Incidents, select the incidents, and use 'Delete' or 'Close' as appropriate. 6. If the rule was created via API or PowerShell, use the same method to disable or remove it (e.g., Remove-AzSentinelAlertRule).
