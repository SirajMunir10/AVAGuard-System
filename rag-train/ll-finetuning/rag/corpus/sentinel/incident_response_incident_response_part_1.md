# Incident Response: Incident Response

**Domain:** Sentinel
**Subdomain:** Incident Response
**Incident Type:** Incident Response

## Scenario / Query
A security analyst in a Microsoft Sentinel environment needs to investigate a high-severity incident generated by an analytics rule. The incident contains multiple alerts and entities. The analyst must use the Microsoft Sentinel investigation graph to visualize relationships and determine the scope of the attack. How can the analyst launch the investigation graph and interpret the entities and links shown?

## Environment Context
- **Tenant Type:** Microsoft 365 E5 with Microsoft Sentinel enabled
- **Configuration:** Sentinel workspace has at least one analytics rule generating incidents, and the user has the 'Microsoft Sentinel Contributor' role.

## Symptoms
- A high-severity incident appears in the Microsoft Sentinel Incidents blade.
- The incident contains multiple alerts and entities (e.g., user accounts, IP addresses, hosts).
- The analyst needs to understand how the entities are connected to determine the attack path.

## Error Codes
N/A

## Root Causes
1. The incident was generated by an analytics rule that correlates multiple data sources.
2. Entities were extracted from the underlying alerts and linked by the Sentinel entity mapping.

## Remediation Steps
1. 1. In the Microsoft Sentinel portal, navigate to 'Incidents' and select the incident.
2. 2. Click 'View full details' to open the incident pane.
3. 3. Select the 'Investigation' tab (or click the 'Investigate' button) to open the investigation graph.
4. 4. Review the graph: entities (represented as nodes) and relationships (represented as edges) show how the entities are connected.
5. 5. Hover over an entity to see its properties and related alerts.
6. 6. Use the graph controls to expand or collapse nodes, zoom, and pan to explore the full scope.
7. 7. Use the 'Entities' tab to see a list of all entities and their types.
8. 8. Document the findings and take appropriate response actions (e.g., isolate affected hosts, disable compromised accounts).

## Validation
After following the remediation steps, the analyst should be able to see the investigation graph with all entities and their relationships, enabling a clear understanding of the incident scope.

## Rollback
Close the investigation graph; no changes are made to the environment by viewing the graph.
