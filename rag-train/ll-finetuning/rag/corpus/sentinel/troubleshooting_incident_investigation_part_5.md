# Troubleshooting: Incident Investigation

**Domain:** Sentinel
**Subdomain:** Incident Investigation
**Incident Type:** Troubleshooting

## Scenario / Query
How do I use the investigation graph in Microsoft Sentinel to deep dive into an incident?

## Environment Context
- **Tenant Type:** Azure
- **Configuration:** Analytics rule must have entity mapping fields configured; incident must be less than 30 days old

## Symptoms
- Unable to investigate an incident using the investigation graph
- Investigation graph does not display entities or relationships

## Error Codes
N/A

## Root Causes
1. Original incident does not include entities because entity mapping fields were not used when setting up the analytics rule
2. Incident is older than 30 days

## Remediation Steps
1. Select an incident, then select Investigate to open the investigation graph
2. Select an entity to open the Entities pane and review information on that entity
3. Ensure the analytics rule used entity mapping fields when it was set up
4. Only investigate incidents that are up to 30 days old

## Validation
1. In Microsoft Sentinel, navigate to Incidents and select an incident that is less than 30 days old and was generated by an analytics rule with entity mapping fields configured.
2. Click 'Investigate' to open the investigation graph.
3. Verify that the graph displays entities and relationships (e.g., users, hosts, IP addresses) connected to the incident.
4. Click on an entity in the graph and confirm that the Entities pane opens and shows relevant information such as timeline, alerts, and bookmarks.
5. If the graph is empty or shows no entities, check the analytics rule that triggered the incident to ensure entity mapping fields (e.g., Account, Host, IP) are properly configured.

## Rollback
1. If the investigation graph still does not display entities or relationships, confirm that the incident is not older than 30 days. If it is, create a new analytics rule with entity mapping fields configured and wait for a new incident to be generated.
2. If the analytics rule lacks entity mapping fields, edit the rule in Analytics > Active rules, select the rule, and click 'Edit'. In the 'Entity mapping' section, add the appropriate entity types (e.g., Account, Host, IP) mapped to the relevant fields in your data source. Save the rule.
3. After updating the rule, allow time for new incidents to be generated with entities. Then repeat the validation steps.
4. If the issue persists, verify that the data connector for the source (e.g., Azure AD, Windows Security Events) is sending the expected fields and that the entity mappings match the field names in the raw logs.
