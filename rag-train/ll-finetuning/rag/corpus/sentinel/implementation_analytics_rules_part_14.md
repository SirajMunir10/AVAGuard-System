# Implementation: Analytics Rules

**Domain:** Sentinel
**Subdomain:** Analytics Rules
**Incident Type:** Implementation

## Scenario / Query
How do I view the results of custom analytics rules in the Defender portal?

## Environment Context
- **Tenant Type:** Azure/Defender
- **Configuration:** Custom analytics rules created in Defender portal

## Remediation Steps
1. Expand Investigation & response in the navigation menu, then Incidents & alerts.
2. View incidents on the Incidents page, where you can triage incidents, investigate them, and remediate the threats.
3. View individual alerts on the Alerts page.

## Validation
1. Navigate to the Microsoft Defender portal (https://security.microsoft.com).
2. Expand 'Investigation & response' in the navigation menu, then select 'Incidents & alerts'.
3. On the Incidents page, verify that incidents generated by the custom analytics rule appear and can be triaged, investigated, and remediated.
4. On the Alerts page, confirm that individual alerts from the custom rule are listed with correct details (e.g., severity, status, associated entities).
5. Optionally, run the following KQL query in Advanced Hunting to confirm alert generation:
   AlertInfo
   | where Title contains '<CustomRuleName>'
   | project Timestamp, AlertId, Title, Severity

## Rollback
1. If the custom analytics rule is causing excessive false positives or performance issues, disable the rule:
   - In the Defender portal, go to 'Investigation & response' > 'Incidents & alerts' > 'Custom detection rules'.
   - Locate the rule, select it, and click 'Disable'.
2. If the rule was misconfigured, edit the rule to correct the query or settings:
   - From the same 'Custom detection rules' page, select the rule and click 'Edit'.
3. To completely remove the rule and its generated alerts:
   - Delete the rule by selecting it and clicking 'Delete'.
   - Note: Deleting a rule does not remove already generated incidents or alerts; those must be manually closed or resolved if needed.
