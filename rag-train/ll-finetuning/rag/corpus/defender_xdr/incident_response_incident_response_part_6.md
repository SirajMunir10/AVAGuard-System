# Incident Response: Incident Response

**Domain:** Defender XDR
**Subdomain:** Incident Response
**Incident Type:** Incident Response

## Scenario / Query
A user reports receiving an email with a malicious attachment that was not blocked by Exchange Online Protection. The security team needs to investigate the incident using Microsoft Defender XDR. How can they identify the email, check for related alerts, and take remediation actions such as soft-deleting the email and blocking the sender?

## Environment Context
- **Tenant Type:** Microsoft 365 E5 with Microsoft Defender for Office 365 Plan 2
- **Configuration:** Exchange Online Protection and Microsoft Defender for Office 365 are enabled. Audit logging is enabled.

## Symptoms
- User received an email with a malicious attachment that was not automatically blocked.
- No alert generated by Exchange Online Protection for the specific email.
- User reported the email manually.

## Error Codes
N/A

## Root Causes
1. The email may have bypassed initial filtering due to a policy misconfiguration or a zero-day threat.
2. The email may have been delivered because the sender or domain was not yet known to threat intelligence.

## Remediation Steps
1. Use Microsoft Defender XDR Advanced Hunting to locate the email: run a query on EmailEvents table filtering by recipient, sender, and subject.
2. Review related alerts in Microsoft 365 Defender portal under Incidents & Alerts.
3. Soft-delete the email using the Threat Explorer or the following PowerShell command (Exchange Online PowerShell): Get-Mailbox -Identity <user> | Search-Mailbox -SearchQuery 'Subject:<subject>' -DeleteContent.
4. Block the sender using the Tenant Allow/Block List in the Microsoft 365 Defender portal.
5. Submit the email attachment to Microsoft for analysis using the Submissions page in Microsoft 365 Defender.

## Validation
Confirm the email is no longer in the user's mailbox and that subsequent emails from the same sender are blocked. Verify that the alert status is updated in the incident.

## Rollback
If the email was incorrectly soft-deleted, recover it using the Recoverable Items folder in Exchange Online or by using the Get-Mailbox and Restore-Mailbox cmdlets.
