4.10.19: Group Policy

Overview
This policy setting determines whether the Windows device is allowed to participate in 
cross-device experiences (continue experiences). 
The recommended state for this setting is: Disabled.

Why This Matters
A cross-device experience is when a system can access app and send messages to 
other devices. In an enterprise managed environment only trusted systems should be 
communicating within the network. Access to any other system should be prohibited.

Audit Steps
Navigate to the UI Path articulated in the Remediation section and confirm it is set as 
prescribed. This policy setting is backed by the following registry location with a 
REG_DWORD value of 0. 
HKLM\SOFTWARE\Policies\Microsoft\Windows\System:EnableCdp

Remediation
To establish the recommended configuration via configuration profiles, set the following 
Settings Catalog path to Disabled. 
Administrative Templates\System\Group Policy\Continue experiences on this 
device

Key Takeaways
- A cross-device experience is when a system can access app and send messages to  other devices.
- In an enterprise managed environment only trusted systems should be  communicating within the network.
- Access to any other system should be prohibited.
