4.6.9.1: (L1) Ensure 'Prohibit installation and configuration of

Overview
You can use this procedure to control a user's ability to install and configure a Network 
Bridge. 
The recommended state for this setting is: Enabled.

Why This Matters
The Network Bridge setting, if enabled, allows users to create a Layer 2 Media Access 
Control (MAC) bridge, enabling them to connect two or more physical network 
segments together. A Network Bridge thus allows a computer that has connections to 
two different networks to share data between those networks. 
In an enterprise managed environment, where there is a need to control network traffic 
to only authorized paths, allowing users to create a Network Bridge increases the risk 
and attack surface from the bridged network.

Audit Steps
Navigate to the UI Path articulated in the Remediation section and confirm it is set as 
prescribed. This policy setting is backed by the following registry location with a 
REG_DWORD value of 0. 
HKLM\SOFTWARE\Policies\Microsoft\Windows\Network 
Connections:NC_AllowNetBridge_NLA

Remediation
To establish the recommended configuration via configuration profiles, set the following 
Settings Catalog path to Enabled. 
Administrative Templates\Network\Network Connections\Prohibit installation 
and configuration of Network Bridge on your DNS domain network
