12.1: (L2) Ensure 'Allow Camera' is set to 'Not allowed'

Overview
This policy setting controls whether the use of Camera devices on the machine are 
permitted. 
The recommended state for this setting is: Not allowed.

Why This Matters
Cameras in a high security environment can pose serious privacy and data exfiltration 
risks - they should be disabled to help mitigate that risk.

Audit Steps
1. Navigate to the following registry location and note the WinningProvider GUID. 
This value confirms under which User GUID the policy is set. 
HKLM\SOFTWARE\Microsoft\PolicyManager\current\device\Camera:AllowCamera_Winni ngProvider 
2. Navigate to the following registry location and confirm the value is set to 0. 
HKLM\SOFTWARE\Microsoft\PolicyManager\Providers\{GUID}\Default\Device\Camera:
AllowCamera

Remediation
To establish the recommended configuration via configuration profiles, set the following 
Settings Catalog path to Not allowed: 
Camera\Allow Camera

Key Takeaways
- Cameras in a high security environment can pose serious privacy and data exfiltration  risks - they should be disabled to help mitigate that risk.
